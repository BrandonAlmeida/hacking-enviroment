Documentação do Ambiente de Hacking Seguro

1. Visão Geral

Este projeto cria um ambiente Docker seguro e modular, projetado para atividades de segurança e hacking. A arquitetura consiste em dois contêineres principais: um gateway de rede (`hacknet-gateway`) e um cliente Kali Linux (`kali-linux`).

O objetivo é forçar todo o tráfego de internet do contêiner Kali e do navegador do host a passar por um túnel VPN (WireGuard), com a opção de usar a rede Tor. Um firewall robusto ("Kill Switch") garante que não haja vazamento de IP caso a conexão VPN falhe.

2. Arquitetura

2.1. Contêineres

  • `hacknet-gateway`: O cérebro da operação. Este contêiner é responsável por:
    • Estabelecer a conexão VPN (WireGuard).
    • Roteamento de todo o tráfego através da VPN.
    • Executar um proxy Tor (Porta 9050).
    • Executar um proxy SOCKS5 (Dante) para o tráfego da VPN (Porta 1080).
    • Implementar um firewall "Kill Switch" para prevenir vazamentos de IP.
  • `kali-linux`: O ambiente do cliente, contendo as ferramentas do Kali Linux para a execução das atividades. Todo o tráfego deste contêiner é roteado através do `hacknet-gateway`.

2.2. Fluxo de Rede

O tráfego segue o seguinte caminho:

Máquina Host (Navegador)  ──> Proxy (1080 ou 9050) ──┐
                                                     ├─> hacknet-gateway ─> WireGuard VPN ─> Internet
Contêiner `kali-linux` ────────────────────────────┘

2.3. Relação entre VPN e Tor (Tor-over-VPN)

É importante entender como o tráfego do Tor interage com a VPN neste ambiente.

A estrutura implementada é a de Tor-over-VPN.

Você → Proxy Tor (Porta 9050) → Gateway → Túnel VPN → Rede Tor → Internet

Como funciona:

  • O contêiner hacknet-gateway força todo o seu tráfego de saída a passar pelo túnel WireGuard (VPN).
  • O serviço do Tor roda dentro deste contêiner.
  • Quando o serviço do Tor tenta se conectar à rede Tor, essa conexão é, ela mesma, capturada pelas regras do sistema e roteada para dentro do túnel VPN.

Vantagem Principal: Seu provedor de internet não consegue ver que você está se conectando à rede Tor, ele apenas vê tráfego criptografado para o seu servidor VPN.

3. Como Usar

3.1. Pré-requisitos

  • Docker
  • Docker Compose

3.2. Iniciando o Ambiente

  1. Escolha a VPN: Abra o arquivo docker-compose.yml e edite a variável de ambiente WG_CONFIG no serviço `hacknet-gateway`. Substitua o valor pelo nome do arquivo .conf da sua configuração WireGuard desejada (localizado em gateway/wireguard/). 
    environment:
  - WG_CONFIG=br-for-wg-001.conf
  2. Construa e inicie os contêineres: Execute o seguinte comando no terminal na raiz do projeto. 
    docker compose up -d --build

3.3. Usando a partir da Máquina Host

Você pode configurar seu navegador ou outras ferramentas para usar os proxies expostos pelo gateway:

  • Proxy SOCKS5 via VPN (Recomendado):
    • Endereço: 127.0.0.1
    • Porta: 1080
  • Proxy SOCKS5 via Tor:
    • Endereço: 127.0.0.1
    • Porta: 9050

Exemplo com curl para verificar o IP (via VPN):

curl --proxy socks5h://127.0.0.1:1080 ipinfo.io/ip

3.4. Usando a partir do Contêiner Kali

  1. Acesse o terminal do Kali: 
    docker exec -it kali-linux /bin/bash
  2. Opção A: Roteamento Total (Recomendado)
    Configure o gateway como a rota padrão. Todo o tráfego do contêiner Kali passará pela VPN. 
    ip route add default via 10.77.10.4
    Para verificar seu IP público: 
    curl ipinfo.io/ip
  3. Opção B: Roteamento por Aplicação (Proxy)
    Use ferramentas como proxychains ou torsocks para rotear o tráfego de aplicações específicas. 
    # Via VPN
proxychains curl ipinfo.io/ip

# Via Tor
torsocks curl ipinfo.io/ip
    Nota: Você precisará configurar o /etc/proxychains.conf para usar o proxy socks5 10.77.10.4 1080.

4. Funcionalidades de Segurança

  • Kill Switch: O `start.sh` configura o `iptables` e `ip6tables` com uma política padrão de DROP para todo o tráfego de saída (`OUTPUT`) e encaminhamento (`FORWARD`). Regras explícitas permitem a saída apenas pela interface da VPN. Se a VPN cair, a conexão com a internet é bloqueada, prevenindo qualquer vazamento do seu IP real.
  • Prevenção de Vazamento de IPv6: O IPv6 é completamente desativado no contêiner `hacknet-gateway` através de sysctl no `docker-compose.yml`. Isso elimina a complexidade do roteamento dual-stack e remove o risco de vazamentos de IP via IPv6, que foi um problema identificado durante o desenvolvimento.
  • Configuração Dinâmica do Proxy: O script `start.sh` configura dinamicamente o servidor proxy Dante (`sockd`) para usar a interface de rede correta (a da VPN) para suas conexões de saída. Isso força o tráfego do proxy a passar pelo túnel seguro, o que foi crucial para corrigir um vazamento de IP.

5. Estrutura do Projeto

  • docker-compose.yml: Orquestra a criação e configuração dos contêineres.
  • gateway/Dockerfile.gateway: Define como construir a imagem do gateway.
  • gateway/start.sh: O coração do gateway. Script de inicialização que configura a VPN, firewall, e serviços.
  • gateway/danted.conf: Template da configuração do proxy Dante.
  • gateway/torrc: Arquivo de configuração do Tor.
  • gateway/wireguard/: Diretório contendo as configurações de conexão da VPN.
  • kali/Dockerfile.kali: Define como construir a imagem do cliente Kali.
